This is the package that contains the code base for the creational design patterns lab.

Introduction

Set the String parameter imagePath in Images to the directory (relative to $home) where the copies of the images are located.

All the names of classes and methods that you as a student have created have to have the prefix 'S_' and the a describing name.
If you change somewhere other than stated make a comment why you have change there and hand in that class as well.

In this lab a number of walls, doors and rooms are combined to a stable using three different creational design patterns. The class that represent the stable is called Stable and to it rooms can be added and to the room four sides can be added. The sides are all subclasses to MapSite please refer to GoF.
The constructor to a room takes a position, x and y, and the constructor to a wall takes a direction inside a room. A door takes two rooms and places itself between them.
Be careful with how you construct the stable, because the last added side in a direction in a room replaces old side in that direction.
It does not have to be the exact same stable created by the different creational design patterns but they all have to fulfill these requirements:
  1. Two ordinary rooms with at least one door.
  2. Five horseboxes with one boxdoor each.
  3. Three rooms in a row that works like a corridor.
  4. Walls everywhere applicable.

Usage

To choose another construction method choose in the menu, to add a horse press the letter 'h' and to control the person represented by a helmet use the arrow keys.

Purpose

By creating the stable using different creational design patterns you see the patterns stengths and weaknesses. And in the future you will be able to choose between them and use the one that fits best in a given situation.

Section A

Readings

The intro pages to creational pattern chapters in GoF p. 81-85, the design pattern Factory Method starting at p. 107 and the lectures.

Instructions

Create a stable according to the requirements using the design pattern Factory Method that uses the classes in the code base e.g. Room and BoxDoor.

Where to edit

Write the body to the method factoryConstruction in StablePanel.

Hints

Look in the method defaultConstruction in StablePanel to see how the classes in the code base is used there.

Section B

Readings

In Gof about Builder on p. 97-107.

Instructions

Create a stable using the design pattern Builder.

Where to edit

Write the body to the method builderConstruction in StablePanel and create one or two own classes.

Hints

Stable is a singleton, adapt the Builder accordingly. The calls to the Builder to build different parts of the stable can be done in the method builderConstruction.

Section C

Readings

The lectures and the chapter about Prototype in GoF on p. 117-126.

Instructions

Create a stable that, as much possible, is created using prototypes and the design pattern Prototype. Look at the requirements on the stable to find suitable prototypes.

Where to edit

Write the body to the method builderConstruction in StablePanel. Create the classes in the Prototype pattern and somewhere to store the created prototypes.

Hints

Use the patterns in the prior labs the create the prototypes, think about how and who creates the prototypes.
Is the prototype storage suitable as a Singleton?
